# PR-02: Freeze ProviderCapabilitiesV1 schema + shared crate re-export

## Goal
Ensure all provider components share the exact same `ProviderCapabilitiesV1` structure as `greentic-messaging` expects, with versioning and stable serde.

## Changes
- Add (or update) a shared crate/module (e.g. `crates/provider-common`) to define:
  - `ProviderCapabilitiesV1`
  - `ProviderLimitsV1`
  - `ProviderMetadataV1` (optional but recommended)
- Ensure types are:
  - `serde::{Serialize, Deserialize}`
  - `schemars::JsonSchema` if your manifests use schemas
- Add JSON schema snapshot test (optional but useful)

## Acceptance criteria
- All providers can import the shared types
- Round-trip JSON test passes
- CI stays green
